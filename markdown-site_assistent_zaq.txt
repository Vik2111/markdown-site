Ниже минимальный путь «поднять своё» на основе этого репо.
 1. Форк и клон
1. Нажми Fork в GitHub на waynesutton/markdown-site.[1]
2. Клонируй форк к себе:  
   git clone https://github.com/<your-username>/markdown-site.git
   cd markdown-site
 2. Установка и локальный запуск
1. Поставь зависимости:  
   npm install
2. Инициализируй Convex (создаст проект и .env.local):  
   npx convex dev
3. Запусти dev‑сервер:  
   npm run dev
4. Открой http://localhost:5173 в браузере.[1]
 3. Базовая настройка под себя
 Автоматическая конфигурация (рекомендуется)
1. Скопируй пример конфига:  
   cp fork-config.json.example fork-config.json
2. Открой fork-config.json и заполни название сайта, URL, описания и т.п.[1]
3. Применить всё сразу:  
   npm run configure
   Это обновит указанные в README ~11 файлов с конфигурацией.[1]
 Ручной вариант (если хочется контроля)
- Открыть FORK_CONFIG.md и пройтись по списку файлов, редактируя их вручную (название сайта, мета‑теги, openapi, ai‑plugin, темы и т.д.).[1]
 4. Добавление контента
1. Посты блога: создать Markdown‑файлы в content/blog/ c front‑matter:  
yaml
   title: "Your Post Title"
   description: "A brief description"
   date: "2025-01-15"
   slug: "your-post-slug"
   published: true
   tags: ["tag1", "tag2"]
   readTime: "5 min read"
   image: "/images/my-header.png"
   excerpt: "Short text for featured cards"
   Your markdown content here...
2. Статические страницы (about, contacts и т.п.) в content/pages/ с похожими метаданными.[1]
 5. Синхронизация контента
- В dev:  
  npm run sync
- В prod (когда уже есть боевой Convex и .env.production.local c VITE_CONVEX_URL):  
  npm run sync:prod
 6. Продакшн‑развёртывание на Netlify
1. Задеплой Convex:  
   npx convex deploy
   Запомни URL вида https://your-deployment.convex.cloud.[1]
2. Подключи репозиторий к Netlify, в настройках билда:  
   - Build command:  
     npm ci --include=dev && npx convex deploy --cmd 'npm run build'  
   - Publish directory: dist.[1]
3. В Netlify добавь переменные окружения:  
   - CONVEX_DEPLOY_KEY — из Convex dashboard.  
   - VITE_CONVEX_URL — прод‑URL Convex.[1]
Если хочешь, дальше можно разобрать конкретно: настройка siteConfig.ts, llms.txt, ai-plugin.json под твой домен и агентов.

как это сделать
---------------
Чтобы «Добавить переменные среды в панель управления Netlify», нужно сделать это в настройках сайта на самом Netlify, а не в коде репозитория.
​
Куда нажимать в Netlify
Зайдите в Netlify и откройте свой сайт (тот, к которому подключен этот форк markdown-site).
​
В меню сайта выберите Site settings.
​
Перейдите в раздел Build & deploy → Environment → Environment variables (иногда просто Environment variables).
​
Что именно добавить
Согласно инструкции репозитория, нужно создать две переменные:
​
CONVEX_DEPLOY_KEY – скопировать из Convex Dashboard:
Convex Dashboard → ваш проект → Project Settings → Deployment key.
​
VITE_CONVEX_URL – рабочий URL Convex, который вы получили после npx convex deploy, например:
https://your-deployment.convex.cloud.
​
В Netlify:
Нажать Add variable / Edit variables.
​
Ввести Name = CONVEX_DEPLOY_KEY, Value = ваш ключ, сохранить.
​
Добавить вторую переменную: Name = VITE_CONVEX_URL, Value = ваш Convex URL, сохранить.
​
После добавления
Сохраните изменения и заново запустите деплой (Redeploy) сайта на Netlify, чтобы билд подхватил новые переменные.
​
